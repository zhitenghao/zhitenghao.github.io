第一章 STL标准库与泛型编程

  1. STL和泛型（Standard Template Library and Generic Programming）
  2. 所谓泛型就是使用模板为主要工具来编写程序。
  3. 本章根据源码分析C++的STL之体系结构，而STL正式泛型编程的最成功的作品，所以本课程事实上就是以STL为标的深层次探讨泛型编程

1.1 C++标准库

  1. C++标准库就是很多的头文件，STL也是标准库中的东西
  2. STL是标准模板库（STL分为六大部分：容器containers、分配器allocators、算法algorithm、迭代器iterators、适配器adapters、仿函数function）
  3. 容器已经解决了内存的问题，不必管，这背后需要分配器来解决
  4. 对容器中的数据进行操作可以通过容器中的一些函数解决，也可以用STL中的算法（algorithm）得到解决，这和面向对象编程有些区别，这是模板编程
  5. 在算法algorithm和容器之间的桥梁是迭代器，迭代器是一种泛化的指针，指向容器中的数据，算法就处理了迭代器指向的数据
  6. 仿函数的作用就像一个函数
  7. 适配器有：容器适配器、迭代器适配器、仿函数适配器
  8. 使用C++标准库中的头文件不需要带.h后缀，#include
  9. 新式的C头文件不带.h，例如：#include
  10. namespace（命名空间）：C++ 命名空间 | 菜鸟教程 (runoob.com)
  11. 新式的头文件内部组件封装于namespace”std”
  12. 旧式头文件内的组件不封装于namespace”std”
  13. 前闭后开区间：==容器.begin()==指向容器的第一个元素， ==容器.end()==指向容器的最后一个元素的下一个位置
  14. 将 ==容器.end()== 进行解引用并不会得到容器的最后一个元素，这是一个危险动作，并不知道解引用以后会得到什么，产生什么后果

代码实例：

#include <iostream>
#include "vector"
#include "functional"
#include "algorithm"
using namespace std;

void test01(){
    cout<<"hello STL"<<endl;
    int arr[6] = {1, 2, 4, 2, 4, 5};
    vector<int, allocator<int>> v1(arr, arr+6);
    cout<<count_if(v1.begin(), v1.end(), not1(bind2nd(less<int>(), 4))); // 大于等于4
    // count_if->algorithm
    // less -> functional
    // bind2nd -> functional
    // not1 -> functional
}

void test02(){
    // 增强for循环的使用
    vector<int> v;
    for (int i: {1, 2, 3, 4, 5, 6}) {
        v.push_back(i);
    }

    cout<<"==================="<<endl;

    for (auto& elem : v) {
        elem*=3; // elem 就是引用,这会修改容器中的数据
        cout<<elem<<endl;
    }

    for (auto elem : v) { // auto 会自动推断数据类型
        cout << elem << endl;
    }
}

int main() {
//    test01();
    test02();
    return 0;
}

1.2 容器的分类及测试

  1. 可以参考C++初级部分的STL相关知识

  2. 序列式容器：Array、Vector、Deque、List（双向列表）、Forward_List（单向列表）

  3. 关联式容器：键值对，有key、value，适合做快速的查找，set/multiset、map/multimap，底层是红黑树，会自动排序

  4. 无序容器：unordered_set，unordered_multiset、unordered_map，unordered_multimap，底层是哈希表

  5. 容器之间的数据结构是不一样的

  6. Array是在C++11引入的，是一个连续的内存空间，无法扩容

  7. Vector可以自动扩容，扩容动作由分配器做，只可以从尾部加入数据

  8. Deque可以在头和尾增加数据

  9. List，链表，不是连续的内存空间，数据之间由指针连接，底层代码是双向环形列表

  10. Forward_List 单向列表

  11. Set/Multiset，内部使用红黑树做的，它的key和value是不分的，multiset的key是可以重复的

  12. Map/Multimap，内部使用红黑树做的，高度平衡二叉树（高度会调整到左右高度几乎相同），这提高了查找效率，multimap的key是可以重复的

  13. 哈希表：数据要根据某个计算放到某个位置，但会发生数据冲突，但是可以解决冲突，解决方式可以采用如下图所示的方式，这个方式是最被推荐的，但是这可能会导致查找数据变慢，因此链表不可以太长，这也是有机制来解决的

1.2.2 容器 vector

vector // 只能从尾部插入数据
vector 是一段连续的内存空间，它的扩容是成倍的扩容，两倍两倍的空间增长，这可能带来内存浪费的问题
vector容器.size(); // 真正的元素的个数
vector容器.front(); // 返回头元素
vector容器.back(); // 返回尾元素
vector容器.data(); // 该容器的首地址
vector容器.capacity(); // 该容器的容量

1.2.3 容器 list

list容器是双向列表，只要内存有空余，就可以放数据
list.size(); // 该容器元素的个数
list.max_size(); // 该容器可以存放多少个数字
list.front(); // 该容器的头元素
list.back(); // 该容器的尾元素
list.sort(); // 该容器自带的sort()，不需要形参;特别注意标准库也有一个sort(); 
// 当容器有自带的sort()的时候，要用自带的函数，其他容器也一样

1.2.4 容器 forward_list

forward_list 也只能在尾部插入元素
forward_list容器.max_size(); 
forward_list容器.front();
forward_list容器.sort(); // 不需要形参
// 该容器没有size(),没有back();

跟forward_list类似，该容器是非标准库的，所以需要包含头文件：#include<ext\slist>

1.2.9 容器 set / multiset

set 和 multiset 的区别就是 set 里面的数据不可以重复，multiset 里面的数据可以重复
其数据结构是红黑树
set/multiset容器.insert(); // 插入数据，放在合适位置，该容器的数据都是有序的
set/multiset容器.size();
set/multiset容器.max_size();
set/multiset容器.find(elem); // 该容器本身就有一个find()函数，但是还有find(形参列表)全局函数，首选自带的函数

map/multimap<key, value>容器.insert( pair<keyType, valueType>(key, value) );

map/multimap<key, value>容器.max_size();
map/multimap<key, value>容器.find(elem); // 同上

1.2.11 容器 unordered_ set 和 unordered_ multiset

unordered_ set; // 该容器底层是哈希表
